## What the brain actually does

```
truth files                                       (signal_history.json, mini_scanner_rules.json, patterns.json)
  ↓ Step 3: brain_derive.py (deterministic)
4 derived views                                   (cohort_health, regime_watch, portfolio_exposure, ground_truth_gaps)
  ↓ Step 4: brain_verify.py (deterministic guards)
candidate proposals                               (cohort_promotion candidates, regime_alert candidates, exposure_warn candidates)
  ↓ Step 5: brain_reason.py (Opus 4.7 LLM gates)
LLM-judged proposals                              (each gated by cohort_promotion_judge / regime_shift_detector / exposure_correlation_analyzer)
  ↓ Step 6: brain_output.py (unified queue + dual-write rule_proposer adapter)
unified_proposals.json + decisions_journal.json
  ↓ Step 7: brain_telegram.py (digest at 22:05 IST)
Telegram digest with /approve, /reject, /explain commands
```

Cost per brain run: ~$0.10 (3 LLM gates × ~$0.03 each). Three-tier rule classification (Tier W/M/Candidate) with Wilson 95% confidence lower bound for sample-size discipline.

## What the brain DOESN'T already have

- **Sub-regime classification** — `regime_watch.json` reports the simple Bear/Choppy/Bull label that comes from `main.py:get_nifty_info`. The brain doesn't currently re-classify the regime itself; it just consumes the label. The conflation described in `04_regime_detector_mechanics.md` and `05_regime_detector_test.md` is upstream of the brain.
- **Day-6 forced-exit fix proposal** — the warning is on cohort_health.json metadata (`r_multiple_under_day6_dominance` warning) but no proposal proposes how to fix exits. This is the D-8 `exit_logic_redesign_v1.md` track which is design-only and undrafted.
- **Per-stock or per-sector kill rules beyond kill_001** — the brain proposes Tier M+ boost_patterns when n exceeds thresholds, but it does not propose new kill_patterns (the kill_rule generation path exists in `brain_output.py` but no candidate has crossed the threshold to fire one).
